# md2pdf

Docker-based tool to convert Markdown files into high-quality PDFs using Python, WeasyPrint, and monospaced typography.

> **Note:** This project was entirely generated by AI.

## Features

- ✅ Markdown → PDF conversion with syntax highlighting
- ✅ Monospaced font (Hack) throughout the document
- ✅ Emoji support (Noto Color Emoji)
- ✅ Auto-fix for lists attached to text
- ✅ Tables and code blocks with automatic wrapping
- ✅ No Node/Chromium or LaTeX dependencies

## Installation

### 1. Build the Docker image

```bash
docker build -t md2pdf .
```

### 2. Make the script executable

```bash
chmod +x md2pdf.sh
```

### 3. (Optional) Add to PATH

```bash
sudo ln -s $(pwd)/md2pdf.sh /usr/local/bin/md2pdf
```

## Usage

### With the wrapper script

```bash
./md2pdf.sh file.md
```

Or, if added to PATH:

```bash
md2pdf file.md
```

### Directly with Docker

```bash
docker run --rm -v $(pwd):/workspace md2pdf file.md
```

## How It Works

1. **entrypoint.py**: Python script that processes Markdown
   - Applies regex to fix attached lists
   - Converts to HTML with extensions (extra, codehilite, toc)
   - Injects custom CSS for WeasyPrint
   - Generates the final PDF

2. **md2pdf.sh**: Bash wrapper that simplifies usage
   - Resolves absolute paths
   - Maps the correct directory in Docker
   - Runs the container

3. **Dockerfile**: Python 3.12 slim image
   - Installs WeasyPrint graphical dependencies
   - Configures Hack and Noto Color Emoji fonts
   - Installs Python libraries (Markdown, WeasyPrint, Pygments)

## Requirements

- Docker installed
- Bash (to use the wrapper script)

## Example

```bash
# Convert a file
md2pdf document.md

# The PDF will be generated as document.pdf in the same directory
```

## Technical Details

### Applied Fixes

- **Attached lists**: Regex injects blank line before non-separated lists
- **Emojis**: Restricted `unicode-range` prevents emojis from breaking numbers
- **Tables**: `table-layout: fixed` + `word-wrap` prevents overflow
- **Code**: `white-space: pre-wrap` forces line breaks in long blocks

### Supported Markdown Extensions

- `extra`: Tables, footnotes, attributes
- `codehilite`: Syntax highlighting with Pygments
- `toc`: Automatic table of contents generation
